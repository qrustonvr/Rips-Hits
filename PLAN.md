# Rips & Hits — TCG Digital Pack Opening Experience

**Lucky Logic internal challenge · 2-week game jam · Mobile-first web app**
**Plan version:** 2.0 · Reconciled against PRD "TCG Pack Opening App" v1.2 (2026-06-08)

The whole point: capture the **weight, texture, sound, and suspense** of a real pack rip on screen. Polish > features. A user should reach for this when they just want the dopamine hit without buying physical cards.

> This plan merges the original game-jam PLAN with the formal PRD v1.2. Where the two genuinely conflict, **both options are kept and flagged** with ⚠️ **CONFLICT** for a decision. Everything else is reconciled into a single source of truth.

---

## Success criteria (from PRD)

- Pick a themed pack, rotate it in 3D, perform the open gesture, and reveal cards one by one — all on a mobile browser in **under 90s from load**.
- Rare reveals feel meaningfully different from commons (visual, audio, pacing).
- **Pre-flip rarity signal** present: the user knows something special is coming before the face shows.
- A non-developer can create a new custom pack in **under 5 minutes** via the wizard.
- Market prices visible on reveal **without a paid API**.
- App loads in **under 3s** on a mid-range Android on LTE.
- Fast-open mode exists; bulk-opening 20 packs takes **under 60s**.

## Decisions (locked)

| Decision | Choice |

| Stack | Vanilla **Three.js (r155+) + Vite**, no framework |
| Renderer | WebGL 2 target, graceful WebGL 1 fallback; DPR capped at `min(devicePixelRatio, 2)` (non-negotiable); rAF loop with delta-time, pause when tab hidden |
| Scope | Full app shell (Home / Collection / Open / Community / Profile) — Open flow gets 90% of the polish budget |

| Card access | All game data behind a shared **`CardSource` adapter**; rest of app sees only normalized rarity tiers |
| Holo | Shader-based holo on card **front only** (per-card foil masks if art allows, else preset patterns) |
| Custom packs | JSON config files in `/packs/`, no code changes to add a pack; resolved by `PackPoolResolver` |
| Fast-open | Built in **from day one**, not retrofitted |
| **Open gesture** | **Tab + pull peel** — grab the corner tab, peel the tear strip across, strip detaches (locked) |

### Open gesture — Tab + pull peel (locked)

Grab the corner tab (haptic tick), peel the tear strip across with your finger. The strip is a separate segmented mesh along the pack top; pull progress (`t` 0→1) drives bone/vertex curl via a simple verlet chain so it sags and curls naturally with your finger. Foil-crinkle audio scrubs with pull velocity, resistance ramps near the end, *pop* on detach — the strip becomes a free rigid body (hand-rolled spring + gravity, **no physics engine**) and flies off. Then cards slide up.

The gesture layer is built around a single `t` (0→1) progress driver, so reveal pacing, audio scrub, haptics, and the failure-recovery states below all read from one value.

## The core loop (every moment matters)

1. **Pack select** — packs fan out on a shelf; pick Pokémon or One Piece. Static list in Phase 1, art added Phase 2–3.
2. **The sealed pack** — 3D `BoxGeometry` pack you can rotate freely (single-finger drag = rotate, two-finger = pinch zoom, limited range). Foil wrap catches light (env-map + normal-mapped crinkle). Subtle idle bob (sine, Y-axis, ~0.05 amplitude). Velocity-based spin inertia decaying over ~1.5s after release.
3. **The tear** — grab the corner tab (haptic tick), peel the strip across. Progress is a single `t` (0→1): the strip curls with your finger (verlet chain), foil crinkle audio scrubs with pull velocity, resistance ramps near the end, *pop* on detach, strip flies off (hand-rolled spring + gravity, **no physics engine**), then cards slide up one at a time.
   - **Failure recovery:** release before `t=0.4` → snap back (spring, 300ms); `t=0.4–0.7` → hold + "keep pulling" nudge; no input for 3s after partial → auto snap-back.
4. **Card-by-card reveal** — cards slide out into a stack/view. Per-card beat sequence below.
5. **The reveal / hit** — back shown first; rarity telegraphed *before* flip (pre-flip glow → slowed flip → audio swell → particle ramp = the "suspense gradient"). Holo + screen-space god rays on hits.
6. **The loop** — results summary (pulls, rarity tally, "new!" badges), cards persist to Collection, one-tap "Rip another."

### Card reveal sequence (beat by beat, from PRD)

1. **Back-face visible** — slides up from the opening (0.3s ease-out).
2. **Pre-flip rarity signal** — back border glows in rarity color. Duration scales with rarity: 0.5s Common → 1.5s Ultra Rare → 2.5s Secret Rare. *The longer the hold, the rarer the card — intentional anticipation.*
3. **Input gate** — tap to flip (Rare+). Commons auto-flip after the signal.
4. **Flip** — 180° on Y; duration scales 0.3s Common → 0.8s Secret Rare.
5. **Rarity reveal event** — at flip midpoint (90°), fire rarity-specific effects (table below).
6. **Post-reveal hold** — face visible, no auto-advance; tap to continue / swipe to dismiss.
7. **Price reveal** — market price fades in below the name after 1.5s.

### Rarity signaling system (5 normalized tiers)

| Tier | Back glow | Flip dur | Screen flash | Particles | Audio stinger | Auto-flip |
|---|---|---|---|---|---|---|
| Common | None | 0.3s | None | None | Soft flip tick | Yes |
| Uncommon | Silver pulse | 0.4s | None | Subtle sparkle | Light chime | Yes |
| Rare | Gold glow | 0.5s | White edge | Gold dust burst | Ascending tone | No — tap |
| Ultra Rare | Prismatic shift | 0.8s | Full-screen white | Prismatic cascade | Multi-note stinger | No — tap |
| Secret Rare | Rainbow cycle + shake | 1.2s | Full-screen + darken | Massive explosion | Dramatic full stinger | No — tap |

Normalized tiers used app-wide: `COMMON`, `UNCOMMON`, `RARE`, `ULTRA_RARE`, `SECRET_RARE`. All rarity normalization lives **inside each adapter*
